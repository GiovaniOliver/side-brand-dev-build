## VIRAL FRAMEWORK SUMMARY

### The 9 Levels (Village Capital 2017)

| Level | Stage Name | Core Goal | Key Validation |
|-------|------------|-----------|----------------|
| **1** | Establishing the Founding Team | Identified problem worth solving | Complementary team skills, direct problem experience |
| **2** | Setting the Vision | Articulate the opportunity | Deep customer understanding, clear value prop |
| **3** | Solidifying Value Proposition | Built product solving real problem | Functional prototype, customers want it |
| **4** | Validating Investable Market | Early signs of big, growing market | Industry needs it, market demand validated |
| **5** | Proving Profitable Business Model | Figured out how to make money | Customers willing to pay, sales pipeline exists |
| **6** | Moving Beyond Early Adopters | Growing past early users | Repeatable/scalable model, users returning |
| **7** | Hitting Product-Market Fit | Nailed solution + sales process | Low CAC, high LTV, predictable growth |
| **8** | Scaling Up | Market leader with traction | Strong brand, hiring culture, proven model |
| **9** | Exit in Sight | Attractive acquisition/IPO candidate | Leadership for scale, defensible model |

### The 5 Evaluation Themes
1. **Team** - Skills, experience, execution ability
2. **Customer** - Understanding, validation, willingness to pay
3. **Business Model** - Revenue, costs, scalability
4. **Investability** - Market size, growth potential, exit path
5. **Goal** - What needs to be proven at this stage

### **Phase 2A: Technical Setup & Infrastructure**
**Duration: 1 day (Week 3, Day 1)**
**Dependencies: None**
**Agent: Platform Engineer or Fullstack Developer**

**Tasks:**
1. Initialize Next.js project
   ```bash
   npx create-next-app@latest [brand-name] --typescript --tailwind --app
   ```
2. Set up Git repository
   - Initialize Git
   - Create .gitignore
   - First commit
3. Create GitHub repository
4. Push to GitHub
5. Install base dependencies
   - Tailwind CSS
   - TypeScript types
   - ESLint/Prettier
6. Create folder structure
   ```
   /app
   /components
   /lib
   /public
   /prisma (if SaaS)
   /sanity (if content brand)
   ```
7. Set up environment variables template (.env.example)

**Deliverables:**
- ✅ Working Next.js project
- ✅ Git version control
- ✅ GitHub repository
- ✅ Base folder structure
- ✅ Development environment ready

---

### **Phase 2B: Core Backend Development**
**Duration: 1 week (Week 3, Day 2 - Week 4)**
**Dependencies: 2A complete**
**Agent: Backend Developer or Python Pro**

**Tasks:**

**For SaaS Brands:**
1. Design database schema
   - User model
   - Content models
   - Relationships
2. Set up Prisma
   ```bash
   npm install prisma @prisma/client
   npx prisma init --datasource-provider sqlite
   ```
3. Create migrations
   ```bash
   npx prisma migrate dev --name init
   ```
4. Set up API routes
   - RESTful endpoints
   - CRUD operations
   - Error handling
5. Authentication (if needed)
   - NextAuth.js setup
   - User sessions
   - Protected routes

**For Content Brands:**
1. Set up Sanity CMS
   ```bash
   npm install sanity @sanity/client next-sanity
   npx sanity init
   ```
2. Define content schemas
   - Blog posts
   - Authors
   - Categories
   - Tags
3. Configure Sanity Studio
4. Set up content fetching
   - GROQ queries
   - Preview mode
   - ISR configuration

**For E-commerce Brands:**
1. Shopify integration
   ```bash
   npm install shopify-buy
   ```
2. Product fetching
3. Cart management
4. Checkout flow

**Deliverables:**
- ✅ Database schema (SaaS)
- ✅ CMS configured (Content)
- ✅ Shopify integrated (E-commerce)
- ✅ API routes working
- ✅ Authentication (if needed)
